Caret: Open-Source AI Development Partner

Caret **aims to integrate** AI agents, local LLMs, and real-time collaboration into a single VS Code extension. It is **an open-source project being built** for developers who want transparent, controllable AI assistance.

Why Caret?

Feeling frustrated by the "black box" nature or limited customization of existing AI coding tools? Caret aims for an experience where developers and AI collaborate as trusted colleagues, providing an environment where you can transparently understand how AI agents work, control them directly according to your needs, and freely connect the LLMs you want.

Key Values (Goals)

  • Developer-controlled orchestration (using .caretrules)
  • Multi-agent & RAG (Alpha supervisor + specialist agents) (Development Goal)
  • Vibe Coding UX (CRDT real-time sync) (Development Goal)
  • Model freedom – OpenAI, Groq, Gemini, or local sLLM (Planned, incl. future support)
    B2B sLLM build & consulting support planned

Core Modules (Target Structure)

Caret is being developed with the following target module structure:

PackageRole
caret-corePrompt & context engine (RAG, cache)
caret-agent4 + 1 modes (Arch / Dev / Rule / Talk / Empty)
caret-ui-liteTerminal & web viewer, CRDT sync
caret-dev-kitLogs, prompt lab, visualization

Roadmap Highlights

Quick Start

git clone https://github.com/aicoding-caret/caret-zero.git
cd caret-zero
pnpm install && pnpm run build:vsix
# VS Code → Extensions → Install from VSIX

Create a .caretrules file in your project root to define modes and rules.
